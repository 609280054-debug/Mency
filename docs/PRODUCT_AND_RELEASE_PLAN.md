# 产品与发布路线

## 产品形态

核心产品定为：

```text
Windows 桌面 AI 调音智能体
```

它不是某一个宿主软件的专用插件，而是常驻在 Windows 上的独立桌面工具。它实时监听用户选择的音频输入源，再结合 AI 对话输出可照抄的调音参数。

## 兼容策略

第一版使用“参数建议兼容”，不直接控制宿主软件。

支持目标：

- Studio Pro 8：用户自用默认目标。
- Studio One 6 / Studio One 7：面向更多录音和唱歌用户。
- 通用 VST 插件链：面向 Cubase、FL Studio、Ableton Live、Reaper 等宿主。

通用输出模块：

- 降噪
- EQ
- 压缩
- De-esser
- 混响
- 延迟
- 限制器

这样第一版不需要分别适配每个软件的私有预设格式，也能给不同用户可执行的调音建议。

## 专业调音流程

用户的核心使用方式是：

```text
干声 + 伴奏 -> AI 调音智能体分析 -> 输出完整调试方案
```

输入方式分两类：

- 实时监听：用户直接在宿主软件里录音或播放，工具监听声卡、虚拟声卡或路由后的声音。这个模式适合边唱边调。
- 文件分析：用户上传干声和伴奏文件，工具先分析录音，再生成调试方案。这个模式适合不方便现场唱歌、或者已经有录音素材时使用。

工具不应该只给零散参数，而要模拟专业调音师流程：

1. 素材诊断：听干声质量、电平、噪声、动态、齿音、浑浊、刺耳，以及人声和伴奏的前后关系。
2. 增益整理：确认输入电平和输出余量，避免还没开始调音就过载。
3. 清理处理：降噪、低切、削掉不需要的低频、控制箱体感和闷感。
4. 音色塑形：用 EQ 处理厚度、清晰度、贴耳感和空气感。
5. 动态控制：用压缩、多段压缩或并行压缩控制忽大忽小的问题。
6. 齿音控制：用 De-esser、动态 EQ 或窄带削减处理 5kHz-10kHz 问题。
7. 空间融合：用混响、延迟、pre-delay 和 wet 比例让人声坐进伴奏。
8. 输出安全：限制器、总输出余量、削波风险检查。
9. 试唱迭代：根据用户反馈继续微调，例如太闷、太刺、太空、压得太死、不够贴耳。

第一版先输出“可照抄的插件链和参数起点”。后续可以增加：

- 更多音频格式上传，例如 mp3、flac、m4a
- 干声和伴奏分轨对比
- 插件链模板库
- 用户常用声线档案
- 风格预设，例如语音厅男歌、抒情歌、贴耳聊天、清亮女声等

## 发布路径

### 1. 桌面 MVP

先完成 Windows 桌面版：

- 音频设备选择
- 实时电平和频段分析
- AI 对话
- 目标宿主选择
- 参数建议输出
- 本地配置和用户偏好保存

### 2. GitHub 仓库

仓库内容：

- 源码
- README
- 截图
- 安装和启动说明
- 发布日志
- issue 模板

### 3. GitHub Releases

发布 Windows 下载包：

- 便携版 zip
- 后续可增加安装版 exe

### 4. 网站下载页

网站用于展示和下载，不承担实时监听核心功能。

页面内容：

- 产品介绍
- UI 截图
- 支持的宿主和插件链
- 下载按钮
- 使用教程
- GitHub 链接
- 更新日志

推荐先用 GitHub Pages，后续再迁移到独立域名或 Vercel。

## 不建议第一版做的事

- 不做纯网页实时调音工具。
- 不自动修改 Studio Pro 8、Studio One 或其他 DAW。
- 不一开始适配各软件的预设文件格式。
- 不做云端账号系统。

这些可以在桌面 MVP 稳定后再逐步做。
